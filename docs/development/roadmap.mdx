## 验证

为了培养一个强大的开发者生态系统，我们计划投资于：

- **参考客户端实现**：通过高质量的 AI 应用展示协议功能
- **合规测试套件**：自动验证客户端、服务器和 SDK 是否正确实现规范

这些工具将帮助开发者自信地实现 MCP，同时确保生态系统内行为的一致性。

## 注册中心

为了让 MCP 充分发挥潜力，我们需要简化的方式来分发和发现 MCP 服务器。

我们计划开发一个[**MCP 注册中心**](https://github.com/orgs/modelcontextprotocol/discussions/159)，以实现集中化的服务器发现和元数据管理。该注册中心将主要作为一个 API 层，供第三方市场和发现服务构建。

## 代理

随着 MCP 越来越多地成为代理工作流的一部分，我们正在探索[改进](https://github.com/modelcontextprotocol/specification/discussions/111)，例如：

- **[代理图](https://github.com/modelcontextprotocol/specification/discussions/94)**：通过命名空间和图感知的通信模式支持复杂的代理拓扑
- **交互式工作流**：通过细粒度权限、标准化交互模式和[与最终用户直接通信的方式](https://github.com/modelcontextprotocol/specification/issues/97)改进人在回路中的体验

## 多模态

支持 MCP 中 AI 能力的完整范围，包括：

- **额外模态**：视频和其他媒体类型
- **[流式传输](https://github.com/modelcontextprotocol/specification/issues/117)**：多部分、分块消息和双向通信，以实现交互式体验

## 治理

我们正在实施优先考虑以下内容的治理结构：

- **社区主导的开发**：培养协作生态系统，让社区成员和 AI 开发者都能参与 MCP 的发展，确保其服务于多样化的应用和用例
- **透明的标准化**：建立清晰的规范贡献流程，同时探索通过行业机构进行正式标准化
